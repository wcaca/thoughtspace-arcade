# thoughtspace-arcade

> 念头空间 - **游戏路线**(储备中,优先推进笔记路线)

## 路线定位
把笔记路线 `wcaca/thoughtspace-notes` 中积累的**念头数据结构 + 几何引擎设计语言**,
包装为**轻量独立游戏**,而不是笔记工具。

## 与笔记路线的关系
| 维度 | 笔记路线 (notes) | 游戏路线 (arcade) |
|---|---|---|
| 目标 | 生产力:记录/整理/觉察/行动 | 游戏体验:探索/收集/成就/叙事 |
| 数据来源 | 用户自己的真实念头 | 预置的"种子念头"+ 玩家游戏内输入 |
| 几何结构 | 全 12 种齐全(Phase 0→4) | 选用 3-5 种最游戏化的(星团/树/螺旋/分形嵌套/管道) |
| 游戏化程度 | 克制(温度/结晶/回顾仪式) | 显性化(等级/任务/成就/剧情) |
| 启动门槛 | 高(用户需积累自己的念头) | 低(打开就能玩) |

## 共享的设计语言(但实现隔离)
两个仓库**不复用代码**,而是复用以下"概念契约":
1. **数据模型**:Note / Thought / Edge / Structure / Action 的字段定义一致
2. **12 种几何结构语义**:G2 星团/G3 树/G8 螺旋/G11 分形嵌套/G12 管道 是共享重点
3. **温度/质量/结晶**三个核心机制的视觉语言保持一致

未来如果要做"笔记数据导出到游戏中",通过 JSON 导出/导入工具桥接,而不是直接共享包。
